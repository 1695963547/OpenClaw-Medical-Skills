# OpenClaw Medical Skills — LangGraph Agent 架构设计

> **定位**：将 869 个生物医学技能集成到 LangGraph ReAct Agent，实现"自动检索 → 按需深读 → 自主规划 → 安全执行"的完整闭环。
> **核心思路**：**语义检索粗筛 → 分层摘要注入 → LLM 按需深读 → Planner 生成计划 → ReAct 自主决策执行 → 结构化错误修正**。
> **技术栈**：LangGraph + LangChain + ChromaDB + Sentence-Transformers

**数据流闭环（8 个节点 → 5 个阶段）：**

1. **auto_retrieve**（入口）：系统强制检索，不等 LLM 决策。用用户原始 query 做 ChromaDB 语义匹配，通过距离阈值（0.7）过滤 → 注入技能**摘要（名称+描述+文件清单）**，不注入 SKILL.md 全文
2. **planner**（紧随其后）：用 `llm_raw` 基于检索结果生成 1~6 步结构化执行计划 → 解析为 `subtasks` 列表
3. **agent**：LLM 动态决策。系统提示词含 OS 环境、workspace 路径、conda 状态、执行计划 → LLM 调用 4 个工具
4. **tools → post_tools**：执行工具 → 结构化错误分类（15+类型）→ 自动依赖安装 → Stuck Detector → 修正建议注入 → 子任务状态更新
5. **循环回 agent** 直至 END、达到 `max_iterations=12`、或 Stuck Detector 触发软着陆

## 一、核心设计决策

| 决策点          | 选择                                                                               | 理由                                                     |
| ------------ | -------------------------------------------------------------------------------- | ------------------------------------------------------ |
| **入口节点**     | `auto_retrieve`（系统强制），非 agent（LLM 决策）                                            | 确保每轮对话必有技能上下文注入；LLM 也可主动调 `retrieve_skills` 做跨领域二次检索   |
| **上下文注入**    | **分层摘要注入**：首次只注入 名称+描述+文件清单（含 SKILL.md 大小），LLM 必须 `read_file("SKILL.md")` 获取全文   | 避免 top_k=8 个 SKILL.md 全文一次性注入导致 token 爆炸（约节省 50%+ 上下文） |
| **Agent 模式** | LangGraph ReAct + Planner + MemorySaver + post_tools 修正                          | LLM 自主决策 + 多步骤规划 + 多轮记忆 + 结构化错误修正                      |
| **工具集**      | `retrieve_skills` + `read_file` + `execute_code` + `update_task_status`          | 检索→读→做→追踪四段分离                                          |
| **执行策略**     | Planner 生成计划 → agent 按计划执行 → update_task_status 追踪进度                             | 避免"中间步骤停止"，多步骤 pipeline 自动引导完成                         |
| **安全隔离**     | 三层防线：黑名单正则 + AST 静态分析（文件白名单/CLI 白名单/危险调用检测）+ venv/conda 双层隔离                     | 进程隔离 + 路径越权防护 + 环境隔离 + 用完即弃                            |
| **多轮对话**     | MemorySaver checkpointer，同一 thread_id 下 messages 跨轮累积                            | Claude Code 做法，Agent 拥有"记忆"                            |
| **流式输出**     | Popen 逐行读取 stdout → 实时 logger 记录                                                 | 长时间任务（BLAST、模型训练）用户可观察进度                               |
| **依赖管理**     | 三层防御：related_skill requirements.txt → auto_install_skill_deps → 代码 import 解析批量安装 | 最大化覆盖，减少 ModuleNotFoundError 重试轮次                      |
| **错误修正**     | 结构化 ErrorType 枚举 + 确定性修正建议注入 + Stuck Detector 循环检测                               | 15+ 错误分类，精准修正，软着陆防无限循环                                 |

### 2.1 AgentState（完整状态定义）

```python
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]   # 对话轮次消息（MemorySaver 持久化）
    selected_skill: str                        # 最终选中的技能 ID
    tool_status: str                           # 工具执行状态（成功/失败/超时/API失败等）
    iteration: int                             # 当前迭代次数
    consecutive_same_result: int               # Stuck Detector 连续相同代码计数
    last_execute_code: str                     # 归一化后的上次执行代码（Stuck Detector）
    force_no_tools: bool                       # Stuck Detector 触发软着陆标志
    plan: str                                  # Planner 生成的执行计划
    subtasks: list[dict]                       # 多步骤任务追踪 [{description, status}]
    retrieved_skills: list[str]                # auto_retrieve 召回技能 ID 列表
```

### 2.3 执行流程详解

**阶段 0：系统自动检索**

`auto_retrieve_node` 在 LLM 决策前强制运行：

- 首次迭代：用用户最新 HumanMessage 作为 query
- 后续迭代：检测 LLM 是否主动调用了 `retrieve_skills(query=...)`，若是则用其 query 做二次检索
- ChromaDB 余弦距离 ≤ 0.7 → 注入 `load_summary()` 返回的摘要（名称+描述+文件清单）
- 注入形式：SystemMessage，含强约束（"优先使用预加载技能"、"必须从 SKILL.md 获取最新 API 地址"）

**阶段 1：生成计划**

`planner_node` 紧接检索后运行：

- 用 `llm_raw`（无工具绑定）生成 1~6 步执行计划
- 简单查询 → 1 步，多步骤 pipeline → 3~6 步
- 计划格式：`1. [步骤类型] 简要描述`，自动解析为 `subtasks` 列表
- 计划注入到 agent_node 的系统提示词中，每轮都可见

**阶段 2：Agent 决策**

`agent_node` 每轮动态构建系统提示词：

- 注入当前 OS、Shell、workspace 绝对路径、conda 可用性 → 避免 LLM 猜测路径
- 注入 Planner 生成的执行计划 + 当前子任务状态
- LLM 自主决定调用哪个工具（4 选 N）

**阶段 3：工具执行**

| 工具                   | 功能         | 关键特性                                                                            |
| -------------------- | ---------- | ------------------------------------------------------------------------------- |
| `retrieve_skills`    | LLM 主动语义检索 | 返回摘要，非 SKILL.md 全文；支持跨领域二次检索                                                    |
| `read_file`          | 读取技能目录文件   | 支持 offset/limit 分页；自动列出可用文件；路径越权防护；读 SKILL.md 末尾自动提醒依赖安装                        |
| `execute_code`       | 安全执行代码     | Python/Bash/R/JavaScript；venv+conda 双层隔离；流式执行；related_skill 自动预装依赖；step_type 标记 |
| `update_task_status` | 更新子任务状态    | LLM 每完成一步更新，post_tools 拦截并更新 AgentState                                         |

**阶段 4：post_tools 修正**

`post_tools_node` 是质量保障核心：

1. **结构化错误分类**：`ErrorType` 枚举（15+ 类型：SCRIPT_ERROR / IMPORT_ERROR / API_AUTH_FAILED / API_RATE_LIMIT / API_NETWORK / API_RESOURCE_NOT_FOUND / API_SERVER_ERROR / TIMEOUT / FILE_NOT_FOUND / PACKAGE_INSTALL_FAILED 等）
2. **自动依赖安装**（三层防御）：
   - 第1层：related_skill → 读取 `requirements.txt` 批量安装
   - 第2层：ImportError → `auto_install_skill_deps()` 补装
   - 第3层：代码 import 解析 → `auto_install_from_code()` 批量安装
3. **确定性错误修正**：ImportError → 自动安装；FileNotFoundError → 提示检查文件列表；KeyError → 提示用 `.get()`；shape mismatch → 提示打印 shape；R 包缺失 → 自动 conda 安装
4. **Stuck Detector**：Jaccard 相似度 ≥ 80% 的连续 execute_code → 注入 Reflection Message + 设 `force_no_tools=True` 软着陆
5. **迭代耗尽修正**：达到 `max_iterations=12` 且尚未完成分析 → 注入截断提醒
6. **子任务状态更新**：拦截 `update_task_status` 调用 → 解析 step_number/status → 更新 AgentState.subtasks

## 三、模块详解

### 3.1 SkillRetriever（src/skill_retriever.py）

- **向量模型**：`paraphrase-multilingual-MiniLM-L12-v2`（中英双语，本地路径优先）
- **索引格式**：`{skill_id}. {description}`，让 embedding 感知技能名称信号
- **增量索引**：新增技能自动入库，已存在跳过
- **模型变更检测**：`_embedding_model.txt` marker 文件记录模型名+索引版本，变更时自动重建 collection
- **两种检索模式**：
  - `retrieve()` → 返回技能列表
  - `retrieve_with_scores()` → 返回技能列表 + 余弦距离（用于阈值过滤）
- **通用 ID 兜底**：`retrieve_skills` 工具中检测 query 是否包含完整技能 ID，若包含则强制加入结果

### 3.2 SkillContext（src/skill_context.py）

- **`load_summary()`**：分层注入，只返回技能描述 + 文件清单（含 SKILL.md 大小），不注入 SKILL.md 全文
- **`load_skeleton()`**：注入 SKILL.md 全文（去 HTML 注释）+ 文件清单摘要
- **`_build_manifest()`**：扫描 examples/、references/、根目录脚本、scripts/ 四个区域
- **`_summarize()`**：智能摘要提取——.py 取首行注释，.md 取首个标题，其他取后缀+大小

### 3.3 CodeExecutor（src/code_executor.py）— 最复杂模块

**支持的 4 种语言与执行配置：**

| 语言         | 执行器                                      | 后缀              | 环境管理       |
| ---------- | ---------------------------------------- | --------------- | ---------- |
| Python     | venv → conda → 系统 Python                 | `.py`           | 双层隔离       |
| Bash       | Git Bash → WSL → PowerShell（Windows 自适应） | `.sh`           | 无          |
| R          | conda R env → 系统 Rscript                 | `.R`            | conda R 隔离 |
| JavaScript | Node.js                                  | `.mjs` / `.cjs` | npm 预安装    |

**三层安全防线：**

| 层   | 方法       | 防护                                                                                                                                                                                |
| --- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 第1层 | 黑名单正则    | `exec/eval/__import__/shutil.rmtree/os.remove/rm -rf/del /f`                                                                                                                      |
| 第2层 | AST 静态分析 | `validate_file_access()` 路径越权检测；`validate_subprocess_calls()` CLI 工具白名单（仅允许 BLAST/bwa/samtools 等 50+ 生物信息学工具）；`validate_dangerous_calls()` 拦截 os.system/popen/spawn/shutil.rmtree |
| 第3层 | 环境隔离     | Per-session venv + conda 双层架构，退出即销毁                                                                                                                                               |

**双层环境隔离架构：**

```
Python 代码中的 # pip install pkg1 pkg2 ...
         │
         ▼
  _classify_packages()
         │
    ┌────┴────┐
    ▼         ▼
纯 Python 包  C 扩展包 (pysam/pybedtools/tables/pybigwig)
    │              │
    ▼              ▼
per-session    conda template 克隆
venv 创建       → _openclaw_bio_template
增量 pip        → per-session conda env
               → conda install C扩展 + pip install 纯Python
```

**依赖安装的三层防御：**

| 层   | 触发时机                            | 机制                                              | 覆盖场景                   |
| --- | ------------------------------- | ----------------------------------------------- | ---------------------- |
| 第1层 | execute_code 有 related_skill 参数 | 读取 `skills/{skill_id}/requirements.txt` → 批量安装  | LLM 按要求填 related_skill |
| 第2层 | post_tools 检测到 ImportError      | `auto_install_skill_deps()` 用 selected_skill 补装 | LLM 忘记填 related_skill  |
| 第3层 | 第2层仍失败                          | `auto_install_from_code()` 解析代码 import 语句批量安装   | 无 selected_skill       |

**R 语言特殊处理：**

- `_extract_r_packages()` 解析 `library()/require()` 中的包名
- 内置 150+ R→conda 包名映射表（`DESeq2 → bioconductor-deseq2`, `ggplot2 → r-ggplot2` 等）
- 静态拦截 `install.packages()/BiocManager::install()`，防止 Windows 源码编译卡死
- conda R 环境按 session 隔离，模板克隆优化

**其他增强特性：**

- `_normalize_paths()`：自动修正 Linux 绝对路径 `/workspace/` → 当前 workspace 路径（负向前瞻避免误触相对路径）
- npm 依赖预安装：JavaScript 代码中的 `require()/import` 包自动 `npm install --prefix`
- `execute_streaming()`：Popen 逐行读取 stdout，实时 `logger.info("[exec] ...")`
- 超时杀进程树：Windows `taskkill /F /T`，Linux `process.kill()`

### 3.4 Agent 工具（src/tools.py）

**`retrieve_skills(query)`**：

- LLM 主动检索，返回摘要（名称+描述+文件清单）
- 通用 ID 兜底：query 含完整 skill ID 时强制加入结果
- 记录检索统计到 `SkillStatsTracker`

**`read_file(skill_id, file_path, limit=300, offset=1)`**：

- 路径越权防护（resolve + startswith 检查）
- 文件不存在 → 自动列出该技能目录下的可用文件
- 目录路径 → 列出目录内文件
- 支持分页读取（offset+limit），大文件不被截断
- 读取 SKILL.md 末页 → 自动附加依赖安装指引

**`execute_code(language, code, related_skill="", step_type="analysis")`**：

- `related_skill` 触发第1层依赖预安装
- `step_type` 帮助 post_tools 判断是否需要继续（preparation/analysis/final）
- 流式执行 → 结构化输出（含 pip_result、api_error 等字段）
- 检测 returncode=0 但含内部错误的边界情况

**`update_task_status(step_number, status, result_summary)`**：

- 被 post_tools 拦截 → 解析 step_number → 更新 AgentState.subtasks
- 主循环中实时展示子任务进度变化（✅/🔄/❌/⬜）

### 3.5 LLM 配置（src/llm_factory.py）

- **配置优先级**：函数参数 > `llm_local_config.py` > 环境变量 > 默认值
- **Profile 系统**：支持 glm/deepseek/deepseek-v4-pro/deepseek-v3 等多个预设
- **Thinking 模式**：DeepSeek 自动启用 thinking；支持 enabled/disabled/auto
- **Gateway 错误诊断**：monkey-patch `_create_chat_result`，choices=null 时打印网关 msg/code
- **兼容性降级**：`timeout` vs `request_timeout` 参数名差异自动适配；tool_choice 不兼容自动退回无工具模式

### 3.6 观测系统

**三层观测：**

| 层次   | 组件                                       | 数据                                                                                                                |
| ---- | ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| 实时日志 | `logging` → 控制台 + `agent_run.log`        | INFO 级摘要 + DEBUG 级完整参数                                                                                            |
| 运行记录 | `agent_run.jsonl`                        | 每轮 JSON 记录：timestamp/thread_id/user_input/retrieved_skills/selected_skill/tool_status/final_answer/duration/steps |
| 技能统计 | `SkillStatsTracker` → `skill_stats.json` | 每技能：检索次数/选中次数/成功/失败/超时/API失败（细分认证/限流/网络）/包安装失败/修正尝试                                                               |
| 链路追踪 | LangSmith（可选）                            | 完整 Agent 决策和执行链路                                                                                                  |

### 3.7 主入口（main.py）

```
启动流程：
1. setup_logging → 控制台 + agent_run.log
2. SkillRetriever 初始化 → 加载 skill_registry.json → ChromaDB 索引
3. build_llm → 解析配置 → ChatOpenAI 客户端
4. setup_langsmith（可选）→ LangSmith 追踪
5. MedicalSkillAgent 初始化 → 构建 LangGraph 图
6. 会话循环：
   - 生成 thread_id（整个对话生命周期不变）
   - set_session_id（venv/conda 隔离上下文）
   - agent.stream() → 解析事件 → 实时展示 + JSONL 记录
   - 每轮结束：stats.save()
7. exit/quit → cleanup_all_sessions() 销毁所有 venv/conda 环境
```

agent:         read_file("bio-blast-searches", "SKILL.md")
tools:         [返回 SKILL.md 全文 + 依赖安装指引]
agent:         read_file("bio-blast-searches", "examples/basic_blast.py")

### 轨迹 3：脚本调用 + 失败回退 + 自动依赖安装

```

  - scripts/basic_grn_inference.py — GRN inference script

                2. [分析计算] 执行基因调控网络推断" (2步)

               → 注入修正建议："已批量安装依赖，请重试原代码"

```

### 轨迹 4：Stuck Detector 触发软着陆

```

           → force_no_tools=True
           → 注入 ReflectionMessage: "你似乎陷入了循环，工具已被禁用，请总结当前进展"
agent: (force_no_tools → llm_raw，无工具绑定)
      "很抱歉，上述方法执行失败。根据已有信息，我建议..."
→ END
```

## 六、设计亮点总结

| 特性                 | 描述                                              |                              |
| ------------------ | ----------------------------------------------- | ---------------------------- |
| **系统强制检索**         | auto_retrieve 在 agent 前运行，不等 LLM 决策             |                              |
| **分层注入**           | 摘要→按需深读，token 节省 50%+                           |                              |
| **Stuck Detector** | Jaccard 相似度检测循环执行，软着陆终止                         |                              |
| **三层依赖防御**         | related_skill → auto_install → import 解析        |                              |
| **双层环境隔离**         | venv（纯 Python）+ conda（C 扩展/R 包），per-session 隔离  |                              |
| **跨平台适配**          | Windows bash/PowerShell/WSL 自适应；路径归一化           |                              |
| **结构化错误分类**        | ErrorType 枚举 15+ 类型 → 精准修正建议注入                  |                              |
| **多步骤导航**          | Planner → subtasks → update_task_status → 进度可视化 |  |
| **观测闭环**           | 日志 + JSONL + skill_stats + LangSmith 四层观测       |                              |
| **LLM 兼容性**        | timeout/request_timeout 自适应；tools 不兼容自动降级       |                              |
